# Умный гайд по страхованию

Интерактивный веб-сервис для кейса I.g.S на IT Purple Hack 2026

Проект помогает подросткам 11–18 лет разобраться, где страховка правда может помочь, а где нет. Это не витрина полисов и не юридический сервис, а понятный обучающий продукт: через реальные жизненные ситуации, короткие объяснения, игровые механики и адаптацию под возраст.

## Что это за продукт

Подросток редко будет читать условия страхования как взрослый договор. Поэтому сервис объясняет логику страхования через знакомые истории:

- разбил смартфон
- сорвалась поездка
- получил травму на тренировке
- потерял доступ к аккаунту
- не попал на концерт
- столкнулся с подозрительным списанием

Вместо сложных формулировок здесь используются:

- сценарии из жизни
- короткие пошаговые разборы
- словарь без канцелярита
- игровые режимы по возрасту
- профиль с прогрессом и памятками

## Для кого сделан проект

В сервисе есть три возрастных режима:

- `11–13 лет`  
  Подача через Лёву, простые аналогии и знакомые бытовые примеры

- `14–15 лет`  
  Больше самостоятельности, но без перегруженного формального языка

- `16–18 лет`  
  Акцент на договор, исключения, документы, сроки и реальные границы покрытия

Возрастной режим влияет на тексты, объяснения, игры и общий тон подачи.

## Что есть внутри

### Сценарии

Сервис включает 9 интерактивных сценариев. Каждый сценарий помогает понять:

- что произошло
- на что смотреть в первую очередь
- где может быть шанс на страховую помощь
- что может всё испортить
- какие доказательства пригодятся

В конце сценария пользователь получает короткую памятку и мини-квиз.

### Основы страхования

Раздел с интерактивными модулями, которые объясняют:

- как вообще работает страхование
- чем риск отличается от страхового случая
- что делать после неприятной ситуации
- как читать исключения

### Словарь

База основных терминов:

- риск
- страховой случай
- покрытие
- исключение
- заявление о случае
- подтверждающие документы
- мошенничество

Для младшей группы словарь объясняется более простым языком и на примерах с Лёвой.

### Игры

В проекте есть отдельный раздел `Игры`. Он сделан так, чтобы не перегружать бесплатный хостинг: вся игровая логика работает в браузере, без тяжёлых библиотек и без лишней нагрузки на сервер.

Для каждого возраста своя механика:

- `11–13 лет` — `Рюкзак Лёвы`  
  Нужно выбрать, что правда поможет после неприятной истории

- `14–15 лет` — `Собери кейс`  
  Нужно быстро отличать событие, доказательство и слабое место кейса

- `16–18 лет` — `Подсвети условие`  
  Нужно читать короткие формулировки как из договора и понимать, где покрытие, где исключение и где список документов

### Профиль

Профиль показывает:

- прогресс по сценариям
- сохранённые памятки
- бейджи
- возрастной режим
- лучший результат в игровых режимах

Если пользователь не вошёл, данные остаются только в браузере.  
Если вошёл, данные синхронизируются с сервером и сохраняются в аккаунте.

## Ключевая идея решения

Проект не заставляет пользователя “выбирать полис”. Он учит задавать правильные вопросы:

- что именно произошло
- есть ли это в условиях
- чем это можно подтвердить
- нет ли исключений
- какие действия нужны сразу

Это делает сервис полезным не только для конкретного набора продуктов, а для общей страховой логики, что соответствует комментариям организаторов по кейсу.

## Запуск локально

### Вариант 1. Через Python

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

После запуска сайт будет доступен по адресу:

```text
http://localhost:8000
```

### Вариант 2. Через Docker

```bash
docker compose up --build
```

После этого приложение тоже будет доступно на:

```text
http://localhost:8000
```

## Переменные окружения

Проект понимает такие переменные:

- `PORT`  
  порт для запуска приложения в облаке

- `SECRET_KEY`  
  секретный ключ Flask

- `DATABASE_URL`  
  строка подключения к БД

Если `DATABASE_URL` не задан, проект работает на локальном `SQLite`.

## Деплой

Проект уже подготовлен к облачному запуску:

- умеет слушать `PORT`
- имеет healthcheck ` /health `
- поддерживает запуск через `Dockerfile`
- может работать с `PostgreSQL`

Публичный демо-инстанс можно поднимать, например, на Render.

## Что важно про продукт

- это не продажа страховок
- это не юридическая консультация
- это не калькулятор выплат
- это обучающий сервис, который помогает подростку понять саму логику

Главная цель проекта: сделать страхование не пугающим и не “взросло-непонятным”, а человеческим и применимым к реальным ситуациям.
